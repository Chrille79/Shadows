# Shadows — project guide

2D platformer. WebGPU client (`client/`) + ASP.NET server (`Shadows.Server/`).
Client uses Vite + TypeScript. All game work lives in `client/src/`.

## Entry points

- `client/index.html` → `src/main.ts` — the game
- `client/editor.html` → `src/editor.ts` — standalone level editor (2D canvas, not WebGPU)

Both are declared as Vite inputs in `client/vite.config.ts`.

## Running

```
cd client
npm run dev          # vite on :5173
npm run build        # tsc + vite build → ../Shadows.Server/wwwroot
npm run extract-tiles  # regenerate src/game/tileTypes.generated.ts from atlas PNGs
```

## Renderer / sprite batcher

`src/engine/renderer.ts` is the WebGPU renderer. `src/engine/spriteRenderer.ts` is
the instanced sprite batcher. It supports multiple `flushWithTexture` calls per
frame by tracking a running `bufferOffset` into the instance buffer — this is
load-bearing, do not reuse the same buffer offset across flushes in one frame
(writeBuffer is immediate but drawIndexed executes at submit time).

World: `WORLD_W = 7680`, `WORLD_H = 1080`, `GAME_W = 1920`, `GAME_H = 1080`.
Camera follows the player with `camX = clamp(player.x - GAME_W/3, 0, WORLD_W - GAME_W)`.

## Stage / platforms

`src/game/stage.ts` — a `Stage` is a list of `Platform { x, y, width, height, type }`
plus `typeResources: Map<string, { bindGroup, srcX/Y, srcW/H, texW/H }>`.

- `createStage()` loads a saved level from `localStorage['shadows:level']` via
  `loadSavedStage()`, falling back to `createDefaultStage()`.
- `loadStageTextures()` only loads atlases referenced by the current stage and
  caches one GPU texture per unique atlas URL.
- `renderStage()` groups platforms by type, computes UVs from each type's atlas
  sub-rect, and flushes once per type. Horizontal tiling is derived from source
  rect aspect: `tileW = p.height * (srcW / srcH)`.

## Tile system

`src/game/tileTypes.ts` — the `TileType` registry. Each has:
- `id` (stable, stored in saved levels)
- `name`, `url`, `solid`
- optional `srcRect: { x, y, w, h }` — atlas sub-rectangle
- `group` — palette grouping key (atlas name for solid tiles, folder name for decorations)

Two sources feed the registry:

1. **Solid tiles** — `src/game/tileTypes.generated.ts` is regenerated by
   `scripts/extract-tiles.mjs` from the 3 atlases in `src/assets/tiles/`
   (`world.png`, `cave.png`, `water.png`). The extractor splits each atlas on a
   **132×132 grid**, skips cells below 2% alpha coverage, then **tightens each
   kept rect to its actual visible bounds** so transparent padding inside a cell
   doesn't push characters off the visible tile. To add new atlases or change
   tiles, edit the `ATLASES` list in the extractor and run `npm run extract-tiles`.
2. **Decorations** — non-solid standalone PNGs under `src/assets/sprites/**/*.png`,
   loaded at runtime via `import.meta.glob`. Folder name becomes the group;
   filename becomes the id/name. Just drop a PNG in a subfolder — no regen needed.

Atlas URLs in the generated file are resolved via
`import.meta.glob('../assets/tiles/*.png', { eager: true, query: '?url' })` —
`new URL(dynamic, import.meta.url)` does **not** work for dynamic paths
(returns `undefined`), only static ones.

## Level editor

`src/editor.ts` paints a tile grid onto a 2D canvas.

- Grid is `TILE = 100` world px per cell (77×11 for the 7680×1080 world).
- Stores type-index + 1 per cell; 0 = empty.
- Palette on left builds canvas thumbnails from each tile's `srcRect`.
- Hover preview shows the selected tile ghosted under a yellow/red cursor frame.
- `Save` writes to `localStorage['shadows:level']`.
- `Play ▶` auto-saves then navigates to `/index.html`.
- `Export/Import JSON` for sharing levels.

Level file format (`version: 2`):

```ts
{
  version: 2,
  tileSize: 100,
  worldWidth: 7680,
  worldHeight: 1080,
  platforms: [{ x, y, width, height, type }]
}
```

Adjacent cells in the same row of the **same type** are merged into one
platform on save, reducing the runtime platform count.

## Conventions

- No `Co-Authored-By` lines in commit messages (user preference, also in global CLAUDE.md).
- Commit messages in English.
- Don't invoke `grep`/`cat`/`find` via Bash — use Grep/Read/Glob tools.
- WebGPU bugs: always check the sprite batcher's `bufferOffset` assumption before adding new flushes.
